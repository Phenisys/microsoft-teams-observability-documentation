Complete reference for the `config.yaml` file. For a guided configuration walk-through, see [Configuring the Collector](/collector/configuration/).

## Validation Pipeline

At startup the collector runs the following validation steps in order:

1. YAML parse
2. Legacy key migration (old keys are automatically remapped)
3. Typed model validation (`AppConfig`)
4. Default feature injection (known feature flags absent from config are added with `enabled: false`)
5. Hybrid validation: JSON Schema + business rules + exporter-specific rules

Unknown top-level keys are tolerated and ignored for forward compatibility.

## Top-Level Structure

**Required sections:** `microsoft_authentication`, `license`, `output`, `collection_config`

**Optional sections:** `telemetry`, `advanced`, `database`

```yaml
microsoft_authentication:
  graph: { ... }
  vaac: { ... }         # optional — required for VAAC features

license:
  filepath: <path>

output:
  dynatrace: { ... }    # optional
  splunk: { ... }       # optional
  otel: [ ... ]         # optional — list
  console: { ... }      # optional

collection_config:
  interval_collection_minutes: 10
  log_level: "INFO"
  features: { ... }
  sites_file: <path>    # optional

telemetry: { ... }      # optional
advanced: { ... }       # optional
database: { ... }       # optional
```

## `microsoft_authentication.graph`

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `tenant_id` | string | Yes | Microsoft Entra tenant ID (GUID) |
| `client_id` | string | Yes | App registration client ID (GUID) |
| `client_secret` | string | Either/or | Client secret value |
| `client_certificate_path` | string | Either/or | Path to PEM file (private + public key) |
| `client_certificate_passphrase` | string | No | Passphrase for encrypted PEM |

<Aside type="note">
Provide either `client_secret` or `client_certificate_path` — not both. RSA certificates only; EC/DSA are not supported.
</Aside>

## `microsoft_authentication.vaac`

Required only when `autoattendant_collection` or `callqueue_collection` feature flags are enabled.

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `tenant_id` | string | Yes | Tenant ID |
| `client_id` | string | Yes | Client ID with VAAC permissions |
| `client_secret` | string | Either/or | Client secret |
| `client_certificate_path` | string | Either/or | Path to PEM file |
| `client_certificate_passphrase` | string | No | PEM passphrase |

## `time_fields_format` Values

Applies to all outputs that support it. Converts all known time fields including `timestamp`.

| Value | Description | Example |
| --- | --- | --- |
| `epoch_ms` | Unix timestamp in milliseconds **(default)** | `1767262830500` |
| `epoch_s` | Unix timestamp in seconds | `1767262830` |
| `epoch_ns` | Unix timestamp in nanoseconds | `1767262830500000000` |
| `iso8601` | ISO 8601 UTC string | `2026-01-01T10:20:30.50Z` |
| `rfc3339` | RFC 3339 UTC string | `2026-01-01T10:20:30.50Z` |

<Aside type="note">
`iso8601` and `rfc3339` are currently equivalent — both produce UTC RFC 3339 strings.
</Aside>

## `collection_config`

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `interval_collection_minutes` | int | No | `10` | Collection interval in minutes |
| `log_level` | string | No | `INFO` | Log verbosity (`DEBUG`, `INFO`, `WARNING`, `ERROR`) |
| `sites_file` | string | No | — | Path to `sites.csv` for subnet-to-site mapping |

## `collection_config.features`

All feature flags default to `false`. Known flags absent from config are auto-injected at startup.

| Feature flag | Default | Description |
| --- | --- | --- |
| `calls_collection.enabled` | `false` | Collect call records and stream details |
| `issues_collection.enabled` | `false` | Collect Microsoft service health announcements |
| `pstn_calls_collection.enabled` | `false` | Collect PSTN call records |
| `direct_routing_calls_collection.enabled` | `false` | Collect Direct Routing records |
| `autoattendant_collection.enabled` | `false` | Collect Auto Attendant data (requires VAAC) |
| `callqueue_collection.enabled` | `false` | Collect Call Queue data (requires VAAC) |

Unknown feature keys are preserved without error.

## `database`

Controls the local SQLite state database (`state.db`) used for deduplication and outbox management. Omitting this section uses all defaults.

| Parameter | Type | Minimum | Default | Description |
| --- | --- | --- | --- | --- |
| `outbox_sent_retention_days` | int | `1` | `7` | Days to keep successfully sent outbox messages |
| `max_processed_objects_per_type` | int | `1000` | `100000` | Max deduplication cache entries per object type; oldest are evicted when exceeded |
| `processed_objects_max_age_hours` | float \| null | — | `null` | Max age (hours) for cache entries. `null` means auto-derived from `max_call_duration_hours × 3` |

**Example — reduce retention to save disk space:**

```yaml
database:
  outbox_sent_retention_days: 2
  max_processed_objects_per_type: 10000
```

## `advanced.quality_thresholds`

Override default call quality classification thresholds. Keep defaults unless you have a specific operational requirement.

```yaml
advanced:
  quality_thresholds:
    audio:
      max_rtt_ms: 500
      max_jitter_ms: 30
```
