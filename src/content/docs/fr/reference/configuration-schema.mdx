Référence complète du fichier `config.yaml`. Pour un guide de configuration pas à pas, voir [Configurer le Collector](/fr/collector/configuration/).

## Pipeline de validation

Au démarrage, le collector exécute les étapes de validation suivantes dans l'ordre :

1. Parsing YAML
2. Migration des clés obsolètes (les anciennes clés sont automatiquement remappées)
3. Validation du modèle typé (`AppConfig`)
4. Injection des feature flags par défaut (les flags connus absents de la config sont ajoutés avec `enabled: false`)
5. Validation hybride : schéma JSON + règles métier + règles spécifiques aux exporters

Les clés de premier niveau inconnues sont tolérées et ignorées pour la compatibilité ascendante.

## Structure de premier niveau

**Sections requises :** `microsoft_authentication`, `license`, `output`, `collection_config`

**Sections optionnelles :** `telemetry`, `advanced`, `database`

```yaml
microsoft_authentication:
  graph: { ... }
  vaac: { ... }         # optionnel — requis pour les fonctionnalités VAAC

license:
  filepath: <chemin>

output:
  dynatrace: { ... }    # optionnel
  splunk: { ... }       # optionnel
  otel: [ ... ]         # optionnel — liste
  console: { ... }      # optionnel

collection_config:
  interval_collection_minutes: 10
  log_level: "INFO"
  features: { ... }
  sites_file: <chemin>  # optionnel

telemetry: { ... }      # optionnel
advanced: { ... }       # optionnel
database: { ... }       # optionnel
```

## `microsoft_authentication.graph`

| Paramètre | Type | Requis | Description |
| --- | --- | --- | --- |
| `tenant_id` | chaîne | Oui | Identifiant du tenant Microsoft Entra (GUID) |
| `client_id` | chaîne | Oui | Identifiant client de l'app registration (GUID) |
| `client_secret` | chaîne | L'un ou l'autre | Valeur du secret client |
| `client_certificate_path` | chaîne | L'un ou l'autre | Chemin vers le fichier PEM (clé privée + publique) |
| `client_certificate_passphrase` | chaîne | Non | Phrase secrète pour le PEM chiffré |

<Aside type="note">
Fournissez soit `client_secret`, soit `client_certificate_path` — pas les deux. Certificats RSA uniquement ; EC/DSA ne sont pas supportés.
</Aside>

## `microsoft_authentication.vaac`

Requis uniquement si les feature flags `autoattendant_collection` ou `callqueue_collection` sont activés.

| Paramètre | Type | Requis | Description |
| --- | --- | --- | --- |
| `tenant_id` | chaîne | Oui | Identifiant du tenant |
| `client_id` | chaîne | Oui | Identifiant client avec permissions VAAC |
| `client_secret` | chaîne | L'un ou l'autre | Secret client |
| `client_certificate_path` | chaîne | L'un ou l'autre | Chemin vers le fichier PEM |
| `client_certificate_passphrase` | chaîne | Non | Phrase secrète PEM |

Chaque entrée de la liste est une destination OTLP indépendante.

| Paramètre | Type | Requis | Description |
| --- | --- | --- | --- |

## Valeurs de `time_fields_format`

S'applique à tous les outputs qui le supportent. Convertit tous les champs de temps connus, y compris `timestamp`.

| Valeur | Description | Exemple |
| --- | --- | --- |
| `epoch_ms` | Timestamp Unix en millisecondes **(par défaut)** | `1767262830500` |
| `epoch_s` | Timestamp Unix en secondes | `1767262830` |
| `epoch_ns` | Timestamp Unix en nanosecondes | `1767262830500000000` |
| `iso8601` | Chaîne UTC ISO 8601 | `2026-01-01T10:20:30.50Z` |
| `rfc3339` | Chaîne UTC RFC 3339 | `2026-01-01T10:20:30.50Z` |

<Aside type="note">
`iso8601` et `rfc3339` sont actuellement équivalents — les deux produisent des chaînes UTC au format RFC 3339.
</Aside>

| Paramètre | Type | Requis | Défaut | Description |
| --- | --- | --- | --- | --- |
| `interval_collection_minutes` | entier | Non | `10` | Intervalle de collecte en minutes |

## `collection_config.features`

Tous les feature flags ont pour valeur par défaut `false`. Les flags connus absents de la config sont auto-injectés au démarrage.

| Feature flag | Défaut | Description |
| --- | --- | --- |
| `calls_collection.enabled` | `false` | Collecter les enregistrements d'appels et les détails des flux |
| `issues_collection.enabled` | `false` | Collecter les annonces de santé du service Microsoft |
| `pstn_calls_collection.enabled` | `false` | Collecter les enregistrements d'appels PSTN |
| `direct_routing_calls_collection.enabled` | `false` | Collecter les enregistrements Direct Routing |
| `autoattendant_collection.enabled` | `false` | Collecter les données Auto Attendant (nécessite VAAC) |
| `callqueue_collection.enabled` | `false` | Collecter les données Call Queue (nécessite VAAC) |

Les clés de feature inconnues sont préservées sans erreur.

## `database`

Contrôle la base de données SQLite d'état local (`state.db`) utilisée pour la déduplication et la gestion de l'outbox. Omettre cette section utilise toutes les valeurs par défaut.

| Paramètre | Type | Minimum | Défaut | Description |
| --- | --- | --- | --- | --- |
| `outbox_sent_retention_days` | entier | `1` | `7` | Jours de rétention des messages outbox envoyés avec succès |
| `max_processed_objects_per_type` | entier | `1000` | `100000` | Nombre max d'entrées de cache de déduplication par type ; les plus anciennes sont évincées si dépassé |
| `processed_objects_max_age_hours` | float \| null | — | `null` | Âge maximum (heures) des entrées de cache. `null` = dérivé automatiquement de `max_call_duration_hours × 3` |

**Exemple — réduire la rétention pour économiser de l'espace disque :**

```yaml
database:
  outbox_sent_retention_days: 2
  max_processed_objects_per_type: 10000
```

## `advanced.quality_thresholds`

Remplacez les seuils de classification de la qualité des appels par défaut. Conservez les valeurs par défaut sauf si vous avez une exigence opérationnelle spécifique.

```yaml
advanced:
  quality_thresholds:
    audio:
      max_rtt_ms: 500
      max_jitter_ms: 30
```
