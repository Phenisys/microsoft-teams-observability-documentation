Cette page est une référence lookup de toutes les permissions Microsoft Graph requises par le collector. Pour le guide de configuration pas à pas, voir [Permissions Azure](/fr/collector/azure-permissions/).

<Aside type="note">
Toutes les permissions listées sont des **permissions Application** (pas Déléguées). Elles doivent être accordées avec le **consentement administrateur** par un Administrateur général ou un Administrateur de rôle privilégié.
</Aside>

## Permissions requises

| Permission | Type | Requis | Objectif |
|---|---|---|---|
| `CallRecords.Read.All` | Application | Oui | Récupérer les enregistrements d'appels Teams, les détails des flux, PSTN et Direct Routing |
| `Reports.Read.All` | Application | Oui | Accéder aux rapports d'activité Microsoft Teams (activité des utilisateurs, usage) |
| `ServiceHealth.Read.All` | Application | Oui | Accéder aux annonces de santé et incidents du service Microsoft 365 |

## Permissions optionnelles (fonctionnalités VAAC)

Les permissions suivantes sont requises uniquement si la collecte Auto Attendant ou Call Queue est activée (flags `autoattendant_collection` / `callqueue_collection`) :

| Permission | Type | Requise pour | Objectif |
|---|---|---|---|
| *(Permissions spécifiques VAAC)* | Application | Auto Attendant, Call Queue | Analytique des applications vocales |

Contactez votre fournisseur pour les permissions VAAC exactes requises dans votre environnement.

## Caractéristiques des permissions

| Attribut | Valeur |
|---|---|
| Type de permission | Application (pas Déléguée) |
| Consentement administrateur | Requis |
| Type de token | Flux client credentials |

## Méthodes d'authentification supportées

| Méthode | Description |
|---|---|
| Secret client | Secret client standard généré dans l'App Registration Azure |
| Certificat (PEM) | Certificat RSA — clé privée + publique dans un seul fichier PEM |

<Aside type="caution">
Les certificats EC/DSA ne sont pas supportés. Utilisez uniquement des certificats RSA.
</Aside>

## Recommandations de sécurité

- Appliquez le **principe du moindre privilège** — accordez uniquement les permissions listées ci-dessus.
- Utilisez de préférence l'**authentification par certificat** plutôt que les secrets clients pour les déploiements en production.
- **Faites tourner les secrets clients** régulièrement et mettez à jour la configuration du collector avant expiration.
- Limitez l'app registration aux **comptes du répertoire de votre organisation uniquement**.
- Surveillez l'activité de l'app registration dans les journaux d'audit Microsoft Entra ID.
