Guide pratique des commandes `ms-teams-agent` les plus utilisées. Pour la référence complète des options, voir [Référence CLI](/fr/reference/cli-reference/).

## Syntaxe des commandes

```bash
ms-teams-agent <commande> [options]
```

| Commande | Objectif |
| --- | --- |
| `run` | Démarrer la collecte et l'export |
| `validate` | Valider la configuration et quitter |
| `test-connection` | Tester l'auth Microsoft et la connectivité backends |
| `service` | Gérer le service systemd Linux |
| `state` | Inspecter ou réinitialiser la base de données d'état |

---

## Commandes de validation

Toujours exécuter ces commandes avant de démarrer la collecte en production :

```bash
# Valider la syntaxe YAML et le schéma
ms-teams-agent validate --config ./config.yaml

# Valider la config + tester l'auth Graph + tester tous les exporters
ms-teams-agent test-connection --config ./config.yaml
```

---

## Démarrer le collector

```bash
# Collecte continue (opération principale)
ms-teams-agent run --config ./config.yaml

# Tester le pipeline sans exporter vers les backends
ms-teams-agent run --config ./config.yaml --dry-run

# Déboguer un appel spécifique
ms-teams-agent run --config ./config.yaml --call-id <CALL_ID> --log-level DEBUG
```

## Gestion du service (Linux)

La plupart des actions de service nécessitent `sudo`.

```bash
# Installer et démarrer en tant que service systemd
sudo ms-teams-agent service enable-service --config /chemin/absolu/config.yaml

# Vérifier le statut du service (instance par défaut)
sudo ms-teams-agent service status

# Vérifier une instance nommée
sudo ms-teams-agent service status --instance prod

# Redémarrer une instance
sudo ms-teams-agent service restart --instance default

# Mettre à jour la config d'une instance en cours (redémarrer seulement si actif)
sudo ms-teams-agent service install-config \
  --config /etc/config.prod.yaml \
  --instance prod \
  --service-restart-if-active

# Supprimer le service entièrement
sudo ms-teams-agent service disable-service
```

Voir [Service Linux](/fr/collector/service/) pour la configuration initiale et la liste complète des actions.

## Inspection de l'état

Le collector maintient une base de données SQLite locale pour suivre les objets traités et éviter les doublons.

```bash
# Afficher l'état actuel (sûr à exécuter pendant que le service tourne)
ms-teams-agent state show

# Exporter l'état vers un fichier JSON
ms-teams-agent state export --output snapshot-etat.json

# Réinitialiser l'état (force un retraitement complet — à utiliser avec précaution en production)
ms-teams-agent state reset
```

## Flux de travail opérationnel recommandé

```bash
# 1. Valider la configuration
ms-teams-agent validate --config ./config.yaml

# 2. Tester la connectivité
ms-teams-agent test-connection --config ./config.yaml

# 3. Démarrer la collecte
ms-teams-agent run --config ./config.yaml
```

Pour les déploiements en service Linux :

```bash
ms-teams-agent validate --config ./config.yaml
sudo ms-teams-agent service enable-service --config /chemin/absolu/config.yaml
sudo ms-teams-agent service status
```
