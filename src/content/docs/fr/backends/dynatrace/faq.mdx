## Licence

**Comment activer le mode live ?**

Installez l'application, finalisez la configuration Dynatrace (Grail, OpenPipeline, Credential Vault), configurez et démarrez le collector, puis appliquez une licence MS Teams Observability valide dans l'écran **Installation & Configuration** de l'application.

**Faut-il redéployer l'application après un changement de licence ?**

En général non. Sauvegardez la configuration et rechargez ou redémarrez l'application si nécessaire.

**Une licence manquante peut-elle provoquer des erreurs de configuration ?**

Oui. Une licence absente ou invalide peut bloquer l'initialisation du tenant et empêcher les sauvegardes de configuration de se terminer correctement.

## Données et visibilité

**Le collector est-il nécessaire en mode démo ?**

Non. Le mode démo utilise des données d'exemple intégrées et est disponible immédiatement après l'installation.

**Pourquoi "No Mapped Calls" apparaît-il dans Sites/Localisations ?**

Le sous-réseau source n'est pas mappé à un site. Mettez à jour `sites.csv` dans la configuration du collector et redémarrez-le. Voir [Configuration du Collector](/fr/collector/configuration/#optional-sitescsv-mapping).

**Peut-on rechercher des appels par identité de l'appelant ou du destinataire ?**

Oui, si ces champs sont présents dans les données indexées pour votre version de l'application.

**Qu'est-ce qu'un "appel vide" ?**

Une session très courte ou interrompue où le média ne s'est pas établi complètement. Une faible proportion d'appels vides est normale.

## Plateforme Dynatrace

**Quelle version de Dynatrace est requise ?**

Dynatrace SaaS avec Grail activé. Dynatrace classique sans Grail n'est pas supporté.

**Dynatrace Managed (on-premise) est-il supporté ?**

Uniquement si l'instance Managed dispose de Grail. Validez avec votre équipe Dynatrace.

**Puis-je utiliser Grail pour restreindre l'accès à des champs sensibles ?**

Oui. Utilisez les fieldsets, les droits basés sur les rôles et le masquage à l'ingestion ou à la requête pour contrôler l'exposition des données sensibles (noms d'utilisateurs, adresses IP).

## Capacité et dimensionnement

**Comment estimer l'impact sur la licence Dynatrace (DPS) ?**

Utilisez les rapports d'activité des utilisateurs Teams pour estimer le volume d'événements quotidiens. Voir [Estimation de licence](/fr/reference/license-estimation/) pour la méthodologie de calcul.

**Puis-je utiliser plusieurs tenants Dynatrace ?**

Le collector peut envoyer vers plusieurs outputs simultanément. Configurez des entrées `dynatrace` supplémentaires dans la section `output` si nécessaire.
