## Modes de fonctionnement

L'application supporte deux modes :

- **Mode démo (par défaut) :** utilise des données d'exemple intégrées. Pas de collector requis. Disponible immédiatement après installation.
- **Mode live :** utilise les données réelles du tenant envoyées par le collector. Nécessite un collector configuré et une licence valide.

## Installer l'application

<Tabs>
  <TabItem label="Dynatrace Hub">
<Steps>
1. Ouvrez **Dynatrace → Hub / Marketplace**.
2. Recherchez **Microsoft Teams Observability**.
3. Installez l'application et accordez les permissions requises.
4. Ouvrez-la depuis **Apps**.
</Steps>
  </TabItem>
  <TabItem label="Upload package ZIP">
<Steps>
1. Téléchargez le dernier package ZIP depuis la [page des releases GitHub](https://github.com/Phenisys/microsoft-teams-observability-dynatrace-app/releases/latest).
2. Dans Dynatrace, ouvrez **Hub → Upload app**.
3. Sélectionnez le fichier ZIP et installez.
4. Vérifiez que le statut de l'application est sain.
</Steps>
  </TabItem>
</Tabs>

## Activer le mode live

Effectuez les étapes suivantes **après** avoir configuré Dynatrace (Grail, OpenPipeline, Credential Vault) et l'output du collector :

<Steps>
1. Ouvrez l'application et accédez à **Installation & Configuration**.
2. Configurez les paramètres du tenant et du backend.
3. Ajoutez et activez votre licence.
4. Sauvegardez et rechargez l'application.
5. Attendez au moins un cycle complet du collector.
</Steps>

<Aside type="caution">
Si la licence est absente ou invalide, l'application peut ne pas initialiser correctement les données du tenant et les sauvegardes de configuration peuvent échouer.
</Aside>

## Liste de vérification post-activation

Après l'activation du mode live, confirmez que :

- La page Vue d'ensemble affiche des événements et métriques réels (non issus de la démo).
- La liste des appels contient des identifiants d'appels réels de votre tenant.
- La page Incidents Microsoft affiche des annonces de service.
- La page Sites/Localisations est renseignée (nécessite le mapping `sites.csv` pour une couverture complète).

En cas de données manquantes, consultez le [Dépannage](/fr/backends/dynatrace/troubleshooting/).
