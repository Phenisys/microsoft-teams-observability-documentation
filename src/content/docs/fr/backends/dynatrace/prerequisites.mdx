Avant de configurer l'intégration Dynatrace, assurez-vous d'avoir les éléments suivants en place.

## Plateforme Dynatrace

- Tenant **Dynatrace SaaS** (Dynatrace Platform — Grail doit être disponible).
- **Grail** activé sur votre tenant (requis pour le stockage des logs et OpenPipeline).

<Aside type="caution">
La gestion classique des logs (sans Grail) n'est pas supportée. Grail doit être actif sur votre tenant Dynatrace.
</Aside>

## Permissions Dynatrace requises

Votre compte Dynatrace doit disposer des droits pour :

| Action | Emplacement |
|---|---|
| Créer et gérer des **buckets de logs** | Paramètres Grail |
| Créer et gérer des pipelines **OpenPipeline** | Paramètres OpenPipeline |
| Créer des entrées dans le **Credential Vault** | Paramètres > Credential Vault |
| Gérer la liste d'hôtes **sortants autorisés** | Paramètres > Préférences |
| Installer des applications depuis le **Hub / Marketplace** | Dynatrace Hub |
| Créer des **tokens API** | Paramètres des tokens d'accès |

## Scopes du token API requis

Créez un token API avec les scopes suivants pour que le collector puisse envoyer des données :

| Scope | Objectif |
|---|---|
| `logs.ingest` | Envoyer des enregistrements de logs à Dynatrace |

<Aside type="note">
Conservez ce token en lieu sûr. Il sera utilisé dans la configuration du collector et peut être stocké dans le Credential Vault.
</Aside>

## Accès au Dynatrace Hub

L'accès au **Dynatrace Hub** est requis pour installer l'application MS Teams Observability. Si l'accès au Hub est restreint dans votre environnement, l'application peut être installée via upload ZIP (voir [Installation](/fr/backends/dynatrace/installation/)).
