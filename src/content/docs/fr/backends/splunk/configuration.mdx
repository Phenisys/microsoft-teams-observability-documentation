## Configuration de l'output du Collector

Ajoutez une entrée `splunk` dans la section `output` de votre `config.yaml` :

```yaml
output:
  splunk:
    enabled: true
    splunk_hec_url: "https://<splunk-host>:8088/services/collector"
    splunk_hec_token: "<hec-token>"
    splunk_ssl_check: true
```

### Paramètres

| Paramètre | Requis | Description |
|---|---|---|
| `enabled` | Oui | Définir à `true` pour activer l'output Splunk |
| `splunk_hec_url` | Oui | URL complète de l'endpoint HEC Splunk |
| `splunk_hec_token` | Oui | Token HEC créé dans Splunk |
| `splunk_ssl_check` | Non | Vérification TLS (par défaut : `true`) |

<Aside type="caution">
Gardez `splunk_ssl_check: true` en production. Ne désactivez la vérification TLS que pour les tests avec des certificats auto-signés, et réactivez-la avant la mise en production.
</Aside>

## Valider la connexion

Après avoir modifié la configuration, exécutez :

```bash
ms-teams-agent validate --config ./config.yaml
ms-teams-agent test-connection --config ./config.yaml
```

## Vérifier l'arrivée des données

Après le premier cycle de collecte réussi :

1. Dans Splunk, lancez une recherche sur l'index `ms_teams` : `index=ms_teams | head 10`
2. Confirmez que des événements sont présents avec les valeurs `source` attendues (`MSTeams_CallRecords_CallMetadata`, etc.).
3. Ouvrez l'application MS Teams Observability Splunk et vérifiez que les dashboards affichent des données.

En cas de problème, consultez le [Dépannage](/fr/backends/splunk/troubleshooting/).
