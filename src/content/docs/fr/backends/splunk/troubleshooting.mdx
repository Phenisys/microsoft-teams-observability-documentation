## Aucune donnée dans l'index ms_teams

**Symptôme :** Aucun événement n'apparaît dans `index=ms_teams`.

**Vérifications :**
1. Exécutez `ms-teams-agent test-connection --config ./config.yaml` et confirmez que l'output Splunk indique un succès.
2. Vérifiez que le token HEC est valide et activé dans Splunk (**Paramètres → Entrées de données → HTTP Event Collector**).
3. Confirmez que l'URL HEC est correcte — elle doit se terminer par `/services/collector`.
4. Vérifiez que l'index `ms_teams` existe et que le token HEC est configuré pour y écrire.
5. Consultez les logs du collector pour détecter des erreurs d'export (`tail -f logs/agent.log`).

## Erreurs d'authentification HEC

**Symptôme :** Les logs du collector affichent `401 Unauthorized` ou `403 Forbidden` sur l'export Splunk.

**Vérifications :**
1. Vérifiez que le token HEC a été copié correctement (pas d'espace en fin de chaîne).
2. Confirmez que le token est activé dans Splunk.
3. Vérifiez que le token dispose des droits d'écriture sur l'index `ms_teams`.

## Erreurs TLS / SSL

**Symptôme :** Les logs du collector affichent des erreurs de certificat ou de handshake TLS.

**Vérifications :**
1. Si vous utilisez un certificat auto-signé sur Splunk, vous pouvez soit :
   - Installer le certificat CA sur l'hôte du collector, soit
   - Définir `splunk_ssl_check: false` (test uniquement — déconseillé en production).
2. Vérifiez que l'URL de l'endpoint HEC Splunk utilise le bon nom d'hôte correspondant au certificat.

## Les tableaux de bord n'affichent aucune donnée

**Symptôme :** Les dashboards Splunk sont vides malgré la présence d'événements dans l'index.

**Vérifications :**
1. Confirmez que les extractions de champs fonctionnent : `index=ms_teams | head 1 | table *` et vérifiez les champs attendus.
2. Vérifiez le parsing des timestamps — assurez-vous que `startDateTime` est parsé correctement.
3. Vérifiez que la plage temporelle des dashboards couvre la fenêtre de collecte des données.

## Mapping des sites non fonctionnel

**Symptôme :** Le dashboard Sites/Localisations affiche "No Mapped Calls" ou des résultats vides.

**Vérifications :**
1. Confirmez que la table de correspondance (`SubnetToSite`) est importée dans Splunk.
2. Vérifiez que `sites.csv` est configuré dans la [Configuration du Collector](/fr/collector/configuration/#optional-sitescsv-mapping).

Pour les problèmes côté collector (authentification Microsoft Graph, erreurs de collecte), consultez le [Dépannage du Collector](/fr/collector/troubleshooting/).
</Aside>
