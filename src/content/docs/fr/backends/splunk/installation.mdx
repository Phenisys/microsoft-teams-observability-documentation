## 1. Installer l'application Splunk

Installez l'application MS Teams Observability sur votre Search Head ou Search Head Cluster Splunk :

- **Splunkbase** : [splunkbase.splunk.com/app/7850](https://splunkbase.splunk.com/app/7850/)

<Aside type="note">
Vérifiez la disponibilité actuelle du package sur Splunkbase pour votre environnement avant de démarrer.
</Aside>

## 2. Créer un input HEC dédié

Configurez un input HTTP Event Collector qui écrit dans l'index `ms_teams`.

<Steps>
1. Accédez à **Paramètres → Entrées de données → HTTP Event Collector**.
2. Créez un nouveau token HEC.
3. Définissez l'index par défaut sur `ms_teams` (créez l'index au préalable s'il n'existe pas).
4. Sauvegardez et copiez le token HEC et l'URL de l'endpoint.
</Steps>

Référence HEC Splunk : [Documentation HEC Splunk](https://docs.splunk.com/Documentation/Splunk/latest/Data/UsetheHTTPEventCollector)

Conservez le **token** et l'**URL de l'endpoint** — ils seront nécessaires pour la [configuration du collector](/fr/backends/splunk/configuration/).

## 3. Configurer le tableau de bord de l'application

L'application Splunk inclut un tableau de bord **MS Teams Configuration** avec des instructions de configuration pas à pas.

Étapes de validation recommandées :

<Steps>
1. Confirmez que les événements arrivent dans l'index `ms_teams`.
2. Ouvrez les tableaux de bord principaux (vue d'ensemble, détail des appels, qualité des sites).
3. Validez les extractions de champs et le parsing des timestamps.
</Steps>

## 4. Optionnel : mapping sites/localisations

Certains tableaux de bord utilisent une table de correspondance sous-réseau/site (par exemple `SubnetToSite`) pour activer l'analyse par site. Importez et maintenez cette table pour améliorer la couverture des dashboards sites/localisations.

Cela correspond à la configuration optionnelle `sites.csv` dans la [Configuration du Collector](/fr/collector/configuration/#optional-sitescsv-mapping).
