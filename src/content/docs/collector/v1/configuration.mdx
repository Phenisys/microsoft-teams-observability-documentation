This page helps you build a valid **v1** `config.yaml` progressively, section by section.

## 1. Configure Microsoft Authentication

Start with `microsoft_authentication`.

```yaml
microsoft_authentication:
    # Microsoft Graph API authentication configuration. Need Reports.Read.All, ServiceHealth.Read.All, CallRecords.Read.All permissions
    microsoft_tenant_id: 'your_tenant_id'
    microsoft_client_id: 'your_client_id'
    microsoft_client_secret: 'your_client_secret'
    # microsoft_client_assertion_type: 'urn:ietf:params:oauth:client-assertion-type:jwt-bearer'
    # microsoft_client_certificate_path: '/path/to/pem'
    # microsoft_client_certificate_passphrase: 'my_passphrase. Delete field if not used'
    microsoft_scope: 'https://graph.microsoft.com/.default'
    microsoft_grant_type: 'client_credentials'
    
    #VAAC Account. Documentation : https://learn.microsoft.com/en-us/microsoftteams/aa-cq-cqd-historical-reports#access-permissions
    microsoft_username: 'your_account'
    microsoft_password: 'your_password'
```

<Aside type="caution">
`CallRecords.Read.All`, `Reports.Read.All`, and `ServiceHealth.Read.All` application permissions are required before startup.
</Aside>

<Aside type="tip">
Use `microsoft_username` and `microsoft_password` only if you enable VAAC features (`ms_teams_autoattendant_collection`, `ms_teams_callqueue_collection`).
</Aside>

## 2. Configure Output Destinations

Add the `output` section and choose where events are sent.

```yaml
output:
    dynatrace:
        enabled: false # Set to true if you want to send data to Dynatrace
        dynatrace_tenant_id: 'your_tenant_id' # Enter your tenant ID only (not the full URL), e.g., https://<tenant_id>.apps.dynatrace.com
        dynatrace_api_token: 'your_token' # Need log.ingest permission
    splunk:
        enabled: false # Set to true if you want to send data to Splunk
        splunk_hec_url: 'hec_url' # Ex: 'https://mysplunk:8088/services/collector/event'
        splunk_hec_token: 'hec_token'
        splunk_ssl_check: false
    console:
        enabled: false # Debug option : Set to true if you want to only test this script without sending any data elsewhere
```

<Aside type="note">
At least one output must be enabled (`dynatrace`, `splunk`, or `console`) or startup will fail.
</Aside>

## 3. Configure Collector Runtime Settings

Now add `collection_config` (logging, license, interval, features).

```yaml
collection_config:
    logging_services: ['dynatrace','splunk','console'] # Available services: dynatrace, splunk, console
    microsoft_max_call_duration_hours: 5
    logfile_log_level: 'INFO' # Be careful with debug mode, there's a lot of written logs and sensitive data!
    # License path format example:
    # - Windows : 'C:\Program Files\Phenisys\license.lic' or '%ProgramFiles%\Phenisys\license.lic'
    # - Linux : '/opt/phenisys/license.lic'
    license_file: '/path/to/license.lic'
    interval_collection_minutes: 1
    features:
        ms_teams_calls_collection:
            enabled: true
        ms_teams_issues_collection:
            enabled: true
        ms_teams_pstn_calls_collection:
            enabled: true
        ms_teams_direct_routing_calls_collection:
            enabled: true
        ms_teams_site_definition:
            enabled: true
            # Site lookup definition from CSV file. Format must be comma separated value with these headers : subnet,site,country,lon,lat
            # Ex : 
            # subnet,site,country,lon,lat
            # 10.1.0.0,Paris_HQ,FR,43.1003,5.8384
            site_definition_file: '/path/to/sites.csv' 
        ms_teams_autoattendant_collection:
            enabled: true
        ms_teams_callqueue_collection:
            enabled: true
```

<Aside type="tip">
Keep `collection_config.logging_services` aligned with the outputs you actually enable.
</Aside>

<Aside type="caution">
`DEBUG` can expose sensitive metadata and generate large logs. Use it only for short troubleshooting windows.
</Aside>

## 4. Optional Site Mapping File (`sites.csv`)

When `ms_teams_site_definition.enabled: true`, `site_definition_file` must point to a CSV with this header order:

```text
subnet,site,country,lon,lat
```

If no valid CSV is loaded, calls appear as unmapped in site views.

## 6. Validate Before Running

```bash
ms-teams-agent validate --config ./config.yaml
```
