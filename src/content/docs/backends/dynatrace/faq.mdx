## Licensing

**How do I activate Live mode?**

Install the app, complete the Dynatrace configuration (Grail, OpenPipeline, Credential Vault), configure and run the collector, then apply a valid MS Teams Observability license in the app's **Installation & Configuration** screen.

**Do I need to redeploy the app after a license change?**

Usually no. Save the configuration and reload or restart the application if prompted.

**Can a missing license cause configuration errors?**

Yes. A missing or invalid license can block tenant initialisation and prevent configuration saves from completing successfully.

## Data and Visibility

**Do I need the collector for Demo mode?**

No. Demo mode uses built-in sample data and is available immediately after installation.

**Why is "No Mapped Calls" appearing in Sites/Locations?**

The originating subnet is not mapped to a site. Update `sites.csv` in the collector configuration and restart the collector. See [Collector Configuration](/collector/configuration/#optional-sitescsv-mapping).

**Can I search calls by caller or callee identity?**

Yes, if those fields are present in the indexed data for your app version.

**What is an "empty call"?**

A very short or interrupted session where media did not fully establish. A small proportion of empty calls is normal.

## Dynatrace Platform

**Which Dynatrace version is required?**

Dynatrace SaaS with Grail enabled. Classic Dynatrace without Grail is not supported.

**Is Dynatrace Managed (on-premise) supported?**

Only if the Managed instance has Grail available. Validate with your Dynatrace account team.

**Can I use Grail to restrict access to sensitive fields?**

Yes. Use fieldsets, role-based access, and masking at ingest or query time to control exposure of sensitive data (user names, IPs).

## Capacity and Sizing

**How do I estimate the Dynatrace license (DPS) impact?**

Use Teams user activity reports to estimate daily event volume. See [License Estimation](/reference/license-estimation/) for the calculation methodology.

**Can I use multiple Dynatrace tenants?**

The collector can send to multiple outputs simultaneously. Configure additional `dynatrace` entries in the `output` section if needed.
