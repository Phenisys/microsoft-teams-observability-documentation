> **Path:** `/users`

## What This Page Shows

The Users page provides per-user analytics across your Microsoft Teams environment. For each user in the selected timeframe, it shows identity, call statistics, network data, device usage, and last activity.

Breadcrumb navigation: `Home > Users`

## Filter Bar

### Filter Field

Advanced filter input with the same syntax style as Calls.

Use this field to isolate a specific user or user pattern before comparing quality/network/device context.

### Timeframe Selector

Scopes which calls are included in user statistics. Stored as `?from=...&to=...`.

### Update Button

Applies filter and timeframe.

When values change, this button applies them and refreshes all table metrics.

## Users Table

Each row represents one user for the selected period. Columns are grouped.

This page helps answer three questions quickly:
- who is impacted,
- how often they are impacted,
- in which network/device context.

### Identity columns

| Column | Sortable | Description |
|--------|----------|-------------|
| ID | No | User unique identifier |
| Name | No | Display name; click opens Calls filtered for this user |
| Email | No | User email |

### Network group

| Column | Sortable | Description |
|--------|----------|-------------|
| IP(s) | No | Observed IP addresses |
| Subnet(s) | No | Observed subnets |
| Protocol(s) | No | UDP/TCP/other chips |

### Calls group

| Column | Sortable | Description |
|--------|----------|-------------|
| Total | Yes | Total calls |
| Good | Yes | Good calls |
| Poor | Yes | Poor calls |
| Unclassified | Yes | Unclassified calls |
| Average Duration | Yes | Mean duration |
| Max Duration | Yes | Longest duration |
| Min Duration | Yes | Shortest duration |

These columns are useful to identify recurrent degradation for specific users.

### Devices group

| Column | Sortable | Description |
|--------|----------|-------------|
| Platform(s) | Yes | Windows/macOS/iOS/Android/Web/Unknown |
| Host(s) | No | Host device names |
| Input(s) | No | Audio input devices |
| Output(s) | No | Audio output devices |

Device columns help identify clusters (same platform/version/peripheral profile across impacted users).

### Additional columns

| Column | Sortable | Description |
|--------|----------|-------------|
| Teams Version | No | Observed client versions |
| Last Seen | Yes | Most recent activity |

## Table Controls

| Control | Description |
|---------|-------------|
| Column visibility | Show/hide columns |
| Column ordering | Reorder columns |
| Line wrap | Toggle wrapping |
| Pagination | 10/20/50/100 rows |

## Empty / Error States

| Situation | Behavior |
|-----------|----------|
| Loading | Spinner / skeleton rows |
| No matching users | Empty table |
| Invalid filter | Update disabled |

## URL Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `filter` | Advanced filter expression | `participant = "John Doe"` |
| `from` | Timeframe start | `now()-7d` |
| `to` | Timeframe end | `now()` |

## Navigation from This Page

| Action | Destination |
|--------|-------------|
| Click user name | <a href={`${import.meta.env.BASE_URL}backends/dynatrace/app/calls/`}>Calls</a> pre-filtered for this user |

<Aside type="note">
Use Users to identify affected people and recurrence, then use Calls/Call Overview for root-cause analysis.
