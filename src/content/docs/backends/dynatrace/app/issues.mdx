> **Path:** `/issues`

## What This Page Shows

The Issues page displays Microsoft service health incidents affecting your tenant, including active and recently resolved issues.

Breadcrumb navigation: `Home > Issues`

## Filter Bar

### Timeframe Selector

Controls which closed issues are shown. Active issues are always shown.

Stored as `?from=...&to=...`.

### Update Button

Applies selected timeframe and refreshes closed issues list.

If you change only the timeframe, this is the action that recalculates the closed-issues list.

## Current Issues

- Displays currently active incidents.
- Header includes active issue count.
- Includes **Open in Notebooks** action.
- Empty state: `No current issues`.

Use this section to confirm whether an active Microsoft event can explain a current quality degradation.

## Closed Issues

- Displays resolved incidents within selected timeframe.
- Includes **Open in Notebooks** action.
- Empty state: `No closed issues`.

Use this section to verify whether a recent issue overlaps your investigation period.

## Issue Card

Card colors reflect severity.

| Severity | Card color | Chip color |
|----------|-----------|-----------|
| Incident | Red/Critical | Red |
| Advisory | Yellow/Warning | Yellow |
| Other | Neutral | Grey |

Card header fields:

| Element | Description |
|---------|-------------|
| Feature name | Affected Microsoft service |
| Issue ID | Incident identifier |
| Start time | Issue start timestamp |
| End time | Closed issues only; approximated |
| Classification | Severity chip |

Card body is expandable and renders detailed incident sections in Markdown.

Expanding a card shows detailed Microsoft-provided information (impact scope, service details, and updates).

## Empty / Error States

| Situation | Behavior |
|-----------|----------|
| Loading | Skeleton cards |
| No current issues | Empty text in current section |
| No closed issues | Empty text in closed section |
| Fetch error | Error state in relevant section |

## URL Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `from` | Timeframe start | `now()-7d` |
| `to` | Timeframe end | `now()` |

<Aside type="note">
Closed issue end times are approximated. Always validate impact and recovery with telemetry from Home, Sites, Calls, and Call Overview.
</Aside>
